Stage 01 · Eligibility & Enrollment
Manual eligibility tracking
  • HR manually monitors service hours & age thresholds
  • Eligibility calculated periodically — not in real time
  • Auto-enrollment triggered reactively after payroll events
Pain: Delayed enrollment · compliance risk from missed eligibility windows
Stage 01 · Eligibility & Enrollment
AI eligibility prediction agent
  • AI reads census and hours data from Omni every cycle
  • Flags who becomes eligible in next 30 / 60 / 90 days
  • Auto-enrollment pre-staged; HR notified in advance automatically
Outcome: Proactive weekly report · zero missed eligibility windows
Stage 02 · Contributions
Payroll file processing & match calculation
  • Employer sends payroll file — Omni "load and go" picks it up
  • Data type errors fail the load — ops team investigates manually
  • Employer match true-up calculated once a year by hand
Pain: File errors cause delays · true-up errors create compliance exposure
Stage 02 · Contributions
AI file validation & true-up agent
  • AI validates payroll file the moment it lands — before load-and-go runs
  • Data errors diagnosed and returned with specific fix instructions in minutes
  • Match true-up calculated continuously — discrepancy flagged same day
Outcome: File errors caught before they fail · true-up always current
Stage 03 · Loans
Manual loan eligibility & document review
  • Participant calls or emails to ask about eligibility — wait for response
  • General purpose: ops team manually runs eligibility check
  • Primary residence: docs uploaded → Omni workflow → Genpact L1 ops manually reviews each document
  • L2 FIS escalates; process restarts if docs are incomplete
Pain: 3–4 days turnaround · fraud risk on uploaded docs · no pre-validation
Stage 03 · Loans
AI loan eligibility & document validation agent (UC-14)
  • Participant self-serves: AI instantly calculates max loan, EMI schedule, and loan-vs-hardship comparison
  • Docs validated at point of WRS upload — completeness + fraud detection before Omni
  • Valid requests fast-tracked to nightly cycle; deficient requests returned with exact requirements
Outcome: 3–4 days → real time · fraud caught at submission · L1 review eliminated
Stage 04 · Hardship Withdrawals
Manual hardship adjudication via Omni ops workflow
  • Participant initiates in WRS — selects reason, uploads documents
  • Omni validates basic data then generates ops workflow task
  • Genpact L1 manually checks: valid IRS safe-harbor reason? Right docs? Amount within limits?
  • Rejected requests sent back — process restarts from beginning
  • Approved request parked for nightly cycle; disbursed next day
Pain: 3–5 days end-to-end · inconsistent decisions · no audit trail on reasoning
Stage 04 · Hardship Withdrawals
AI hardship adjudication agent — BUILT (UC-16)
  • AI retrieves plan-specific hardship provisions via semantic search
  • Applies 4-check framework: IRS safe-harbor reason · vested balance · amount limit · documentation
  • Returns APPROVED / DENIED / ESCALATE in under 5 seconds with full citation and rationale
  • Immutable audit log: every decision linked to exact plan section and IRS regulation
Outcome: 3–5 days → 5 seconds · consistent decisions · full audit trail on every case
Stage 05 · Distributions & RMD
Manual RMD identification & calculation
  • 59½ withdrawals processed via same Omni manual ops review flow
  • RMD: plan admin manually identifies participants aged 72½+ from census
  • IRS life-expectancy table looked up by hand per participant
  • Missed RMDs trigger 25% excise tax — discovered after the fact
Pain: Manual IRS table lookup at scale · missed RMDs a real compliance risk
Stage 05 · Distributions & RMD
AI RMD identification & calculation agent (UC-17)
  • AI scans all participant records every cycle — identifies anyone at or past RMD age
  • Calculates RMD per participant using IRS Uniform Lifetime Table automatically
  • Checks prior-year distributions — flags shortfall risk before deadline
  • Generates audit-ready compliance report; alerts administrators proactively
Outcome: Zero missed RMDs · 25% excise tax risk eliminated · automated at any plan scale
Stage 06 · Compliance Testing
Manual ADP/ACP, HCE, and 402(g) testing
  • Compensation and deferral data pulled from Omni nightly reports manually
  • HCE classification from prior-year compensation — error-prone at scale
  • ADP/ACP ratios calculated in spreadsheets
  • Test failures discovered late — corrective distributions often rushed
Pain: Highest manual effort · late discovery of failures compounds penalties
Stage 06 · Compliance Testing
AI compliance testing agent
  • AI reads Omni nightly reports — runs ADP/ACP, HCE, and 402(g) monitoring every cycle
  • Test failures flagged in real time — corrective action begins immediately
  • HCE classification verified automatically from compensation data, no spreadsheets
  • Compliance calendar managed by AI — deadlines tracked, escalations triggered automatically
Outcome: Continuous compliance vs annual crunch · failures caught early · penalties avoided
Stage 07 · Beneficiary & QDRO
Manual beneficiary management & death claims
  • Beneficiary designations updated manually on request only
  • No proactive communication to participants about current designations
  • On death: multiple claimants often dispute — legal review, potential litigation
  • QDRO interpretation done by legal team — slow and expensive
Pain: Disputes discovered only on death · QDRO interpretation delays weeks
Stage 07 · Beneficiary & QDRO
AI beneficiary transparency & QDRO agent
  • Monthly AI report to each participant: current balance + named beneficiaries + % each receives
  • Participant acknowledgement captured — documented trail reduces death-claim disputes
  • QDRO: AI reads order language, checks qualification criteria, flags ambiguities before legal review
Outcome: Disputes reduced before they arise · QDRO pre-screened in hours not weeks
Stage 08 · Tax Reporting
Manual 1099-R, 5500, and SAR assembly
  • Distribution events aggregated manually at year-end
  • 1099-R codes applied by hand per distribution type
  • Underpayment risk not calculated until filing — surprises for participants
  • 5500 data compiled from multiple Omni reports — labour-intensive
Pain: Year-end crunch · underpayment risk invisible until too late
Stage 08 · Tax Reporting
AI tax reporting intelligence agent
  • AI aggregates all distribution events from Omni reports throughout the year
  • Correct 1099-R code applied automatically per event type
  • Underpayment risk calculated in real time — participants alerted before Q4
  • 5500 signal data pre-compiled — audit support ready on demand
Outcome: Year-round accuracy · underpayment risk visible early · audit package ready
Stage 09 · Nightly Cycle Exceptions
Next-day manual exception review
  • Nightly cycle runs and generates exception report
  • Ops team reviews the report next business morning
  • Each exception researched and resolved manually
  • Recurring errors not identified as patterns
Pain: Next-day turnaround · recurring errors missed · no pattern detection
Stage 09 · Nightly Cycle Exceptions
AI nightly exception triage agent
  • Immediately after nightly cycle, AI reads exception report and triages every item
  • Common failures auto-resolved or routed to correct team with diagnosis
  • Recurring errors identified as patterns — root-cause fix recommended
  • Summary delivered before ops team starts next morning
Outcome: Same-night resolution · patterns surfaced · no next-day surprise backlog
